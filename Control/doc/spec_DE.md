Entität: Steuerung  
==================  
[Offene Lizenz](https://github.com/smart-data-models//dataModel.EnergyCIM/blob/master/Control/LICENSE.md)  
Globale Beschreibung: **Abgeleitet aus CIM-Datenmodellen. Control wird für die Überwachung/Gerätesteuerung verwendet. Sie repräsentiert Steuerausgänge, die zur Änderung des Zustands in einem Prozess verwendet werden, z. B. Schließen oder Öffnen eines Schalters, eines Sollwerts oder eines Befehls zum Anheben oder Absenken.**  

## Liste der Eigenschaften  

- `PowerSystemResource`: Die Reglerausgänge, mit denen ein Regelgerät, z. B. die Magnetisierung einer Synchronmaschine oder eines Kondensatorbatterie-Stellglieds, tatsächlich geregelt wird. Voreinstellung: Keine  - `address`: Die Postanschrift  - `alternateName`: Ein alternativer Name für diesen Artikel  - `areaServed`: Das geografische Gebiet, in dem eine Dienstleistung oder ein angebotener Artikel erbracht wird  - `controlType`: Gibt den Typ der Steuerung an, z. B. BreakerOn/Off, GeneratorVoltageSetPoint, TieLineFlow usw. Der ControlType.name muss unter allen angegebenen Typen eindeutig sein und den Typ beschreiben. Voreinstellung: ''  - `dataProvider`: Eine Folge von Zeichen, die den Anbieter der harmonisierten Dateneinheit identifiziert.  - `dateCreated`: Zeitstempel der Entitätserstellung. Dieser wird normalerweise von der Speicherplattform zugewiesen.  - `dateModified`: Zeitstempel der letzten Änderung der Entität. Dieser wird in der Regel von der Speicherplattform vergeben.  - `description`: Eine Beschreibung dieses Artikels  - `id`: Eindeutiger Bezeichner der Entität  - `location`:   - `name`: Der Name dieses Elements.  - `operationInProgress`: Zeigt an, dass ein Client gerade Steuerbefehle sendet, die noch nicht abgeschlossen sind. Voreinstellung: False  - `owner`: Eine Liste mit einer JSON-kodierten Zeichenfolge, die auf die eindeutigen Ids der Eigentümer verweist  - `seeAlso`: Liste von uri, die auf zusätzliche Ressourcen über das Element verweist  - `source`: Eine Folge von Zeichen, die die ursprüngliche Quelle der Entitätsdaten als URL angibt. Empfohlen wird der voll qualifizierte Domänenname des Quellanbieters oder die URL zum Quellobjekt.  - `timeStamp`: Das letzte Mal, dass ein Steuerausgang gesendet wurde. Standard: ''  - `type`: NGSI-Typ. Es muss Steuerung sein  - `unitMultiplier`: Der Einheitenmultiplikator der kontrollierten Menge. Voreinstellung: Keine  - `unitSymbol`: Die Maßeinheit der kontrollierten Menge. Voreinstellung: Keine    
Erforderliche Eigenschaften  
Dieses Datenmodell ist eine direkte Umsetzung des Common Information Model (CIM), das durch die Norm IEC61970 spezifiziert ist, in Smart Data Models. Die Python-Klassen, auf denen dieses Modell basiert, wurden von diesen Einrichtungen entwickelt Institut für Automatisierung komplexer Stromversorgungssysteme (ACS), EON Energy Research Center (EONERC) und RWTH Aachen, Deutschland. einige Eigenschaften können einen falschen Typ haben. Dies war der Fall, bitte melden Sie einen Fehler oder senden Sie eine E-Mail an alberto.abella@fiware.org
