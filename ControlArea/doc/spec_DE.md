Globale Beschreibung: **Abgeleitet aus CIM-Datenmodellen. Eine Regelzone ist eine Gruppierung von Erzeugungseinheiten und/oder Lasten und ein Abschnitt von Verbindungsleitungen (als Klemmen), die für eine Vielzahl von Zwecken verwendet werden kann, einschließlich der automatischen Erzeugungssteuerung, der Spezifikation für die Steuerung des Austauschs von Leistungsflusslösungsgebieten und der Eingabe in die Lastprognose.   Beachten Sie, dass eine beliebige Anzahl von sich überschneidenden Regelzonenspezifikationen dem physikalischen Modell überlagert werden kann.**  

## Liste der Eigenschaften  

- `ControlAreaGeneratingUnit`: Die Spezifikation der Erzeugungseinheit für den Kontrollbereich. Standard: 'Liste'  - `EnergyArea`: Die Energiefläche, die aus dieser Regelzonenangabe prognostiziert wird. Voreinstellung: Keine  - `TieFlow`: Die mit dem Kontrollbereich verbundenen Bindungsströme. Standard: 'Liste'  - `address`: Die Postanschrift  - `alternateName`: Ein alternativer Name für diesen Artikel  - `areaServed`: Das geografische Gebiet, in dem eine Dienstleistung oder ein angebotener Artikel erbracht wird  - `dataProvider`: Eine Folge von Zeichen zur Identifizierung des Anbieters der harmonisierten Dateneinheit.  - `dateCreated`: Zeitstempel der Entitätserstellung. Dieser wird in der Regel von der Speicherplattform zugewiesen.  - `dateModified`: Zeitstempel der letzten Änderung der Entität. Dieser wird in der Regel von der Speicherplattform vergeben.  - `description`: Eine Beschreibung dieses Artikels  - `id`: Eindeutiger Bezeichner der Entität  - `location`: Geojson-Referenz auf das Element. Es kann Punkt, LineString, Polygon, MultiPoint, MultiLineString oder MultiPolygon sein  - `name`: Der Name dieses Artikels.  - `netInterchange`: Der angegebene positive Nettodurchfluss in die Regelzone, d. h. ein positives Vorzeichen bedeutet einen Zustrom in die Regelzone. Voreinstellung: 0,0  - `owner`: Eine Liste mit einer JSON-kodierten Zeichenfolge, die auf die eindeutigen Kennungen der Eigentümer verweist  - `pTolerance`: Wirkleistungs-Netzwerktoleranz Standard: 0.0  - `seeAlso`: Liste von URLs, die auf zusätzliche Ressourcen zu dem Artikel verweisen  - `source`: Eine Folge von Zeichen, die die ursprüngliche Quelle der Entitätsdaten als URL angibt. Empfohlen wird der voll qualifizierte Domänenname des Quellanbieters oder die URL des Quellobjekts.  - `type`: Die primäre Art der Regelzonendefinition wird verwendet, um zu bestimmen, ob diese für die automatische Erzeugungssteuerung, für die Planung der Austauschsteuerung oder für andere Zwecke verwendet wird.   Eine Regelzone, die mit dem primären Typ der automatischen Erzeugungssteuerung angegeben wurde, kann dennoch prognostiziert und als Austauschzone in der Leistungsflussanalyse verwendet werden. Voreinstellung: Keine
