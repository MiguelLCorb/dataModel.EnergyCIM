Globale Beschreibung: **Abgeleitet aus CIM-Datenmodellen. Eine Regelzone ist eine Gruppierung von Erzeugungseinheiten und/oder Lasten und eine Schnittmenge von Verbindungsleitungen (als Klemmen), die für eine Vielzahl von Zwecken verwendet werden kann, einschließlich automatischer Erzeugungssteuerung, Spezifikation der Leistungsflusslösungsgebietsaustauschsteuerung und Eingabe für Lastprognosen.   Beachten Sie, dass eine beliebige Anzahl von sich überschneidenden Regelzonen-Spezifikationen dem physikalischen Modell überlagert werden kann.**  

## Liste der Eigenschaften  

- `ControlAreaGeneratingUnit`: Die Spezifikation der Erzeugungseinheit für den Steuerbereich. Standard: 'Liste'  - `EnergyArea`: Die Energiefläche, die aus dieser Regelzonenvorgabe prognostiziert wird. Voreinstellung: Keine  - `TieFlow`: Die mit der Regelzone verbundenen Verbindungsflüsse. Standard: 'Liste'  - `address`: Die Postanschrift  - `alternateName`: Ein alternativer Name für diesen Artikel  - `areaServed`: Das geografische Gebiet, in dem eine Dienstleistung oder ein angebotener Artikel erbracht wird  - `dataProvider`: Eine Folge von Zeichen, die den Anbieter der harmonisierten Dateneinheit identifiziert.  - `dateCreated`: Zeitstempel der Entitätserstellung. Voreinstellung: 0,0  - `owner`: Eine Liste mit einer JSON-kodierten Zeichenfolge, die auf die eindeutigen Ids der Eigentümer verweist  - `pTolerance`: Wirkleistungs-Netzwechseltoleranz Default: 0.0  - `seeAlso`: Liste von uri, die auf zusätzliche Ressourcen über das Element verweist  - `source`: Eine Folge von Zeichen, die die ursprüngliche Quelle der Entitätsdaten als URL angibt. Empfohlen wird der voll qualifizierte Domänenname des Quellanbieters oder die URL zum Quellobjekt.  - `type`: Der primäre Typ der Regelzonendefinition wird verwendet, um zu bestimmen, ob diese für die automatische Erzeugungssteuerung, für die Planung der Austauschsteuerung oder für andere Zwecke verwendet wird.   Eine Regelzone, die mit dem primären Typ der automatischen Erzeugungssteuerung angegeben wurde, könnte dennoch prognostiziert und als Austauschzone in der Leistungsflussanalyse verwendet werden.
