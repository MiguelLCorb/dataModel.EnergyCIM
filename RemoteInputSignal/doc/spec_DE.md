Entität: RemoteInputSignal  
==========================  
[Offene Lizenz](https://github.com/smart-data-models//dataModel.EnergyCIM/blob/master/RemoteInputSignal/LICENSE.md)  
Globale Beschreibung: **Abgeleitet aus CIM-Datenmodellen. Unterstützt die Verbindung zu einer Klemme, die mit einem Fernbus verbunden ist, von dem ein Eingangssignal eines bestimmten Typs kommt.**  

## Liste der Eigenschaften  

- `DiscontinuousExcitationControlDynamics`: Modell der diskontinuierlichen Erregungssteuerung mit diesem Remote-Eingangssignal. Voreinstellung: Keine  - `PFVArControllerType1Dynamics`: Modell des Leistungsfaktors oder VAr-Reglers Typ I, das dieses Remote-Eingangssignal verwendet. Voreinstellung: Keine  - `PowerSystemStabilizerDynamics`: Modell des Netzstabilisators, das dieses Remote-Eingangssignal verwendet. Voreinstellung: Keine  - `Terminal`: Entfernte Klemme, mit der dieses Eingangssignal verknüpft ist. Voreinstellung: Keine  - `UnderexcitationLimiterDynamics`: Untererregungsbegrenzermodell mit diesem Remote-Eingangssignal. Voreinstellung: Keine  - `VoltageCompensatorDynamics`: Spannungskompensatormodell, das dieses Remote-Eingangssignal verwendet. Voreinstellung: Keine  - `WindPlantDynamics`: Das Fernsignal, mit dem dieses Kraftwerk verknüpft ist. Voreinstellung: Keine  - `WindTurbineType1or2Dynamics`: Modell des Windgenerators Typ 1 oder Typ 2 mit diesem Remote-Eingangssignal. Voreinstellung: Keine  - `WindTurbineType3or4Dynamics`: Remote-Eingangssignal, das von diesen Windturbinenmodellen Typ 3 oder 4 verwendet wird. Voreinstellung: Keine  - `address`: Die Postanschrift  - `alternateName`: Ein alternativer Name für diesen Artikel  - `areaServed`: Das geografische Gebiet, in dem eine Dienstleistung oder ein angebotener Artikel erbracht wird  - `dataProvider`: Eine Folge von Zeichen, die den Anbieter der harmonisierten Dateneinheit identifiziert.  - `dateCreated`: Zeitstempel der Entitätserstellung. Dieser wird normalerweise von der Speicherplattform zugewiesen.  - `dateModified`: Zeitstempel der letzten Änderung der Entität. Dieser wird in der Regel von der Speicherplattform vergeben.  - `description`: Eine Beschreibung dieses Artikels  - `id`: Eindeutiger Bezeichner der Entität  - `location`:   - `name`: Der Name dieses Elements.  - `owner`: Eine Liste mit einer JSON-kodierten Zeichenfolge, die auf die eindeutigen Ids der Eigentümer verweist  - `remoteSignalType`: Typ des Eingangssignals. Voreinstellung: Keine  - `seeAlso`: Liste von uri, die auf zusätzliche Ressourcen über das Element verweist  - `source`: Eine Folge von Zeichen, die die ursprüngliche Quelle der Entitätsdaten als URL angibt. Empfohlen wird der voll qualifizierte Domänenname des Quellanbieters oder die URL zum Quellobjekt.  - `type`: NGSI-Typ. Es muss RemoteInputSignal sein    
Erforderliche Eigenschaften  
Dieses Datenmodell ist eine direkte Umsetzung des Common Information Model (CIM), das durch die Norm IEC61970 spezifiziert ist, in Smart Data Models. Die Python-Klassen, auf denen dieses Modell basiert, wurden von diesen Einrichtungen entwickelt Institut für Automatisierung komplexer Stromversorgungssysteme (ACS), EON Energy Research Center (EONERC) und RWTH Aachen, Deutschland. einige Eigenschaften können einen falschen Typ haben. Dies war der Fall, bitte melden Sie einen Fehler oder senden Sie eine E-Mail an alberto.abella@fiware.org
