Entität: PositionPoint  
======================  
[Offene Lizenz](https://github.com/smart-data-models//dataModel.EnergyCIM/blob/master/PositionPoint/LICENSE.md)  
Globale Beschreibung: **Abgeleitet aus CIM-Datenmodellen. Satz räumlicher Koordinaten, die einen Punkt bestimmen, definiert in dem in 'Location.CoordinateSystem' angegebenen Koordinatensystem. Verwenden Sie eine einzelne Positionspunktinstanz, um einen punktorientierten Ort zu beschreiben. Verwenden Sie eine Folge von Positionspunkten, um ein linienorientiertes Objekt (physischer Standort von nicht punktorientierten Objekten wie Kabeln oder Leitungen) oder einen Bereich eines Objekts (wie ein Umspannwerk oder eine geografische Zone - in diesem Fall müssen der erste und der letzte Positionspunkt dieselben Werte haben) zu beschreiben.  

## Liste der Eigenschaften  

- `Location`: Ort, der durch diesen Positionspunkt beschrieben wird. Voreinstellung: Keine  - `address`: Die Postanschrift  - `alternateName`: Ein alternativer Name für diesen Artikel  - `areaServed`: Das geografische Gebiet, in dem eine Dienstleistung oder ein angebotener Artikel erbracht wird  - `dataProvider`: Eine Folge von Zeichen, die den Anbieter der harmonisierten Dateneinheit identifiziert.  - `dateCreated`: Zeitstempel der Entitätserstellung. Dieser wird normalerweise von der Speicherplattform zugewiesen.  - `dateModified`: Zeitstempel der letzten Änderung der Entität. Dieser wird in der Regel von der Speicherplattform vergeben.  - `description`: Eine Beschreibung dieses Artikels  - `id`: Eindeutiger Bezeichner der Entität  - `location`:   - `name`: Der Name dieses Elements.  - `owner`: Eine Liste mit einer JSON-kodierten Zeichenfolge, die auf die eindeutigen Ids der Eigentümer verweist  - `seeAlso`: Liste von uri, die auf zusätzliche Ressourcen über das Element verweist  - `sequenceNumber`: Null-relative Sequenznummer dieses Punktes innerhalb einer Serie von Punkten. Voreinstellung: 0  - `source`: Eine Folge von Zeichen, die die ursprüngliche Quelle der Entitätsdaten als URL angibt. Empfohlen wird der voll qualifizierte Domänenname des Quellanbieters oder die URL zum Quellobjekt.  - `type`: NGSI-Typ. Es muss PositionPoint sein  - `xPosition`: Position der X-Achse. Standard: ''  - `yPosition`: Position der Y-Achse. Standard: ''  - `zPosition`: (falls zutreffend) Position der Z-Achse. Standard: ''    
Erforderliche Eigenschaften  
Dieses Datenmodell ist eine direkte Umsetzung des Common Information Model (CIM), das durch die Norm IEC61970 spezifiziert ist, in Smart Data Models. Die Python-Klassen, auf denen dieses Modell basiert, wurden von diesen Einrichtungen entwickelt Institut für Automatisierung komplexer Stromversorgungssysteme (ACS), EON Energy Research Center (EONERC) und RWTH Aachen, Deutschland. einige Eigenschaften können einen falschen Typ haben. Dies war der Fall, bitte melden Sie einen Fehler oder senden Sie eine E-Mail an alberto.abella@fiware.org
